### 操作流程

| 步骤 | 按钮 | 说明 |
|------|------|------|
| ① | **2. 编译 APP** | Keil 编译 APP → `output/bin/AMKN8639_APP.bin` |
| ② | **1. 编译 BL + 打包** | 编译 BL + 拼接 → `output/bin/AMKN8639_SX.bin` |

| 快速 | **一键: 编译+烧录** | ②+③ 一次性完成 |

### OTA 升级 (串口)

1. 板子正常通电 (APP 运行中)
2. 串口填 `COM46`，点 **OTA升级**
3. 自动: 发送 AT+OTA → 进入 BL → 擦除 → 写入 → 启动
4. 1231KB 约 2.6 分钟

## 四、命令行编译指令

### 编译 APP (Keil MDK)
```bash
cd targets/AMKN8639
"D:\keil5\UV4\UV4.exe" -j0 -b AMKN8639.uvprojx
```
产物: `output/bin/AMKN8639_APP.bin`

### 编译 BL + 打包 SX.bin
```bash
cd targets/AMKN8639/bootloader
powershell -ExecutionPolicy Bypass -File build.ps1
```
产物:
- `bootloader/bootloader.bin` (2060 bytes)
- `output/bin/AMKN8639_SX.bin` (1359 KB)

### 烧录 (ST-Link)
```bash
# 擦除 + 烧录 + 复位
"D:\stmp\bin\STM32_Programmer_CLI.exe" -c port=SWD freq=4000 -e all --skipErase -w output/bin/AMKN8639_SX.bin 0x08000000 -rst
```

### OTA 升级 (命令行)
```bash
# 全自动
python scripts/ota_host.py COM46 auto output/bin/AMKN8639_APP.bin

# 手动 (先复位板子)
python scripts/ota_host.py COM46 flash output/bin/AMKN8639_APP.bin

# 查询 BL 信息
python scripts/ota_host.py COM46 info
```

## 七、内存布局

```
0x08000000 ┌──────────────┐
           │  BL (128KB)  │  Sector 0 — 仅 BL 自升级时擦除
0x08020000 ├──────────────┤
           │  APP (~1.2M) │  Sector 1-10 — OTA 可更新
0x081E0000 ├──────────────┤
           │  BL 暂存区    │  128KB — BL 升级用
0x08200000 └──────────────┘
```

## 八、XCP-lite 协议 (BL V7.2)

| 命令 | 数据 | 说明 |
|------|------|------|
| `0x01` | `uint32 addr` | 擦除扇区 |
| `0x02` | `uint32 addr + N×uint32 data` | 批量写入 (最多 60 字) |
| `0x03` | 无 | 启动 APP |
| `0x04` | 无 | 查询 BL 版本/APP 状态 |
| `0x05` | 无 | LED 闪烁测试 |

**包格式:** `[0xAA][LEN][CMD][DATA...][CKSUM]`  
**ACK:** `[0xAA][0x02][STATUS][CKSUM]` (STATUS: 0=成功)

## 九、OTA 升级流程

### PC 升级
```
PC ──USB转串口──> STM32 UART1 (PA9/PA10, 115200)
  │
  ├─ 发送 "AT+OTA\r\n" → APP 写 BKP 魔法值 → 复位
  ├─ BL 启动 → 检测魔法值 → 进入 XCP 模式
  ├─ XCP: 擦除 Sector 1-10 (~20s)
  ├─ XCP: 批量写入 1231KB (~145s, 60字/包)
  └─ XCP: 启动新 APP
```

### RK3568 远程升级
```
云端 ──HTTP下载──> RK3568 ──UART──> STM32
                    │
                    ├─ python3 rk_ota_agent.py --uart /dev/ttyS1 --fw http://server/app.bin
                    └─ 自动完成: 触发 → 擦除 → 写入 → 启动
```

## 十、常见问题

| 问题 | 解决 |
|------|------|
| BL 编译报错 | 确认 ARMCC v5 路径: `D:\keil5\ARM\ARMCC\bin\armcc.exe` |
| 烧录失败 "erase memory" | 用 `mode=HOTPLUG` 或先 mass erase 再写 |
| OTA 擦除超时 | 确认 BL V7.2 (擦除超时 3s)，旧版只有 78ms |
| 串口无输出 | 检查 COM 口编号，板子是否已通电 |
| 编码报错 GBK | 用 `bl_tool.py` 或 `ota_host.py`，已内置 UTF-8 修复 |
| 烧录后板子不启动 | 必须用 `-e all --skipErase -w` (先全擦再写) |

## 十、OTA 源码 — 给 RK3568 集成用

### 10.1 源码清单（全部在 `BL_BUILD/scripts/`）

| 文件 | 用途 | 运行平台 |
|------|------|----------|
| `ota_host.py` | PC 命令行 OTA 工具，支持 `auto/flash/flash-bl/info/boot` | Windows/Linux PC |
| `rk_ota_agent.py` | RK3568 远程 OTA Agent，支持 HTTP 下载 + 多字批量写入 | RK3568 (Linux) |

### 10.2 协议文档 (XCP-lite)

**物理层:** UART 115200-8-N-1  
**包格式:** `[0xAA] [LEN] [CMD] [DATA…] [CKSUM]`  
**CKSUM:** XOR(CMD, DATA_bytes)  
**ACK格式:** `[0xAA] [0x02] [STATUS] [CKSUM]`

**命令:**

| CMD | 名称 | DATA | ACK STATUS |
|-----|------|------|------------|
| 0x01 | 擦除扇区 | 4B 扇区地址 (LE) | 0=成功 |
| 0x02 | 批量写入 | 4B 起始地址 + N×4B words (最多60字/包) | 0=成功 |
| 0x03 | 启动APP | 无 | 0=成功 |
| 0x04 | 查询信息 | 无 | BL 版本 + APP 状态 |
| 0x05 | LED测试 | 无 | 0=成功 |

### 10.3 RK3568 集成示例

```python
# RK3568 端调用示例
from rk_ota_agent import OTAAgent, load_firmware

agent = OTAAgent('/dev/ttyS1', 115200)
fw = load_firmware('/data/AMKN8639_APP.bin')

# 升级 APP
agent.update(fw)                    # 自动: 进BL -> 擦 -> 写 -> 启动

# 升级 BL (写入暂存区，触发自升级)
agent.enter_bl_mode()
agent.erase(BL_TEMP, len(fw_bl))
agent.write(fw_bl, BL_TEMP)
agent._close()
```

### 10.4 关键参数

| 参数 | 值 |
|------|-----|
| APP_BASE | `0x08020000` |
| BL_TEMP (暂存) | `0x081E0000` |
| 扇区大小 | 128 KB |
| 批量写入 | 60 words/pkt (240B) |
| 波特率 | 115200 |
| 实测速度 | ~7.9 KB/s (~2.6min/1231KB APP) |
