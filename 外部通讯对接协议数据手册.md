## 1. 概述 (Overview)

### 1.1 文档范围

本手册覆盖 **BMS 单片机对外的两条通讯链路**:

```
┌─────────────────────────────────────────────────────────────┐
│                    BMS 单片机 (STM32F103C8T6)                 │
│                                                             │
│   UART1 (CSP v1.0)                                          │
│     ↑↓ AT+XXX / $BMS_STATUS / $EVT,CHG=... / #BIN_LOG...   │
│                                                             │
│   USART2 / SWD (内部调试用, 不在本手册范围)                  │
└─────────────────────────────────────────────────────────────┘
       ↑↓                                  ↑↓
   [外部设备 A]                          [Web 上位机 v3.18]
   (充电器主控/4G 图传)                  (后端 wss + 浏览器 ws)
                                          ↓
                                    [Master 浏览器 WebSerial]
                                          ↓
                                       [UART1 ←→ 同一物理口]
```

**关键事实**: Web 上位机的 Master 浏览器最终也走 UART1, **与外部设备 A 共用同一物理串口** (一时刻只能一个占用).

# 第一部分: UART 协议 (固件 v3.6 CSP v1.0)

## 2. 物理层 (Physical Layer)

| 项目 | 规格 |
|------|------|
| 串口 | USART1 (TTL 3.3V) |
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 校验位 | None |
| 停止位 | 1 |
| 流控 | None |
| 帧格式 | ASCII 文本, `\r\n` 结尾 |
| 电平 | 3.3V TTL (切勿直接接 RS-232 ±12V) |

### 2.1 引脚定义

| 单片机引脚 | 功能 | 外部设备连接 |
|------------|------|--------------|
| PA9 (USART1_TX) | 单片机发送 | 接外部设备 RX |
| PA10 (USART1_RX) | 单片机接收 | 接外部设备 TX |
| GND | 共地 | **必须** 共地 |

### 2.2 隔离建议

工业环境强烈建议加 **数字隔离器** (ADuM1411 / ISO1050):
- 隔离器供电: 单片机侧 3.3V, 外部设备侧独立供电
- 隔离耐压: ≥ 2.5kV (基本) / ≥ 4kV (推荐)

## 3. 协议 (Protocol)

外部设备通过 AT 命令集 + 事件推送与单片机通信. 所有命令以 `\r\n` 结尾.

### 3.1 命令总览 (按功能分组)

#### 系统控制 (5 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+HELP` | 双向 | 显示所有可用命令 |
| `AT+VER` | 双向 | 查询固件版本号 |
| `AT+RST` | 双向 | 恢复出厂设置 (清除所有用户配置) |
| `AT+SAVE` / `AT+SAV` | 双向 | 保存配置到 Flash (掉电不丢) |
| `AT+REBOOT` | 双向 | 软复位 MCU (NVIC_SystemReset) |

#### 状态查询 (4 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+VM?` | 双向 | 查询双电压 VW/VB/VM |
| `AT+MOS?` | 双向 | 查询 MOS 状态 0/1 |
| `AT+FAN?` | 双向 | 查询风扇状态 0/1/2 |
| `AT+TM?` | 双向 | 查询 4 路 NTC 温度 |
| `AT+CFG` | 双向 | 查询完整系统配置 |
| `AT+STATUS` | 双向 | 主动发一帧 NMEA `$BMS_STATUS` |
| `AT+ID` | 双向 | 查询设备 ID |
| `AT+CAL?` | 双向 | 查询电压校准系数 |

#### 控制类 (8 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+MOS=0/1` | 双向 | 关闭/开启 MOS (受 Rule A/B/C 约束) |
| `AT+FAN=0/1/2` | 双向 | 风扇: 0=关 1=开 2=自动 |
| `AT+FANMD=0/1` | 双向 | 风扇粘性模式 0=软 1=硬 (需 `AT+SAVE` 持久化) |
| `AT+LED1..5=0/1` | 双向 | 控制 LED1-5 |
| `AT+BTEST=0/1` | 双向 | LED 全开测试模式 |
| `AT+BIND=0/1` | 双向 | 进入/退出对频模式 |
| `AT+FWU` | 双向 | 进入固件升级模式 (YModem) |
| `AT+CHG=0/1` | 双向 | **外部充电协议**: 开始/停止充电 (受 Rule C 约束) |
| `AT+CHGHB` | 双向 | **外部充电协议**: 纯心跳保活 (不刷 SESSION ID) |
| `AT+CHGTO=<ms>` | 双向 | **外部充电协议**: 配置看门狗超时 (0=禁用) |
| `AT+CHGTO?` | 双向 | **外部充电协议**: 查询看门狗配置 |
| `AT+CHG?` | 双向 | **外部充电协议**: 查询充电状态 |
| `AT+CHGRST` | 双向 | **外部充电协议**: 强制重置 (调试) |

#### 校准类 (5 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+VOFF=x.xx` | 双向 | 电压校准偏移 (范围 -1.0~+1.0V) |
| `AT+VOFF?` | 双向 | 查询电压偏移 |
| `AT+CAL=x.xxxx` | 双向 | 校准系数 (0.5~2.0) |
| `AT+T1=±x.x` ~ `AT+T4=±x.x` | 双向 | NTC 1-4 温度偏移 |
| `AT+T?` | 双向 | 查询所有温度偏移 |

#### 配置类 (3 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+WARN=<v>` | 双向 | 告警电压 (默认 12.8V) |
| `AT+CRIT=<v>` | 双向 | 临界电压 (默认 11.8V) |
| `AT+REC=<v>` | 双向 | 恢复电压 (默认 10.5V) |

#### 调试/EEPROM (5 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+EEP` | 双向 | 进入 EEPROM 读写模式 (页 0-15) |
| `AT+LOGCT` | 双向 | 查询事件日志统计 |
| `AT+LOGRD` | 双向 | 读出所有事件日志 |
| `AT+LOGCLEAR` | 双向 | 清空事件日志 |
| `AT+FORMAT=<password>` | 双向 | 恢复出厂 + 清 EEPROM (密码 `MOSPOWER08`) |
| `AT+TESTLOG` | 双向 | 测试日志写入 |

### 3.2 $BMS_STATUS 主动帧 (5Hz)

单片机每 200ms 主动推一帧 NMEA 风格状态:

```
$BMS_STATUS,Seq=242,VW=14.39,VB=14.41,VM=14.50,MOS=1,SOC=85,T1=25.3,T2=24.8,T3=NC,T4=23.1,FAN=2,L1=1,L2=1,L3=0,L4=0,L5=0,P1=OK,P2=OK,CS=IDLE\r\n
```

| 字段 | 取值 | 说明 |
|------|------|------|
| `Seq` | 000-255 (uint8) | 序列号, 单调递增 (回绕检测) |
| `VW` | float (V) | 负载电压 (PMOS 之后) |
| `VB` | float (V) | 电池电压 (PMOS 之前) |
| `VM` | float (V) | 主板电压 (内部) |
| `MOS` | 0/1 | 当前 MOS 状态 |
| `SOC` | 0-100 (%) | 估算的电池剩余容量 |
| `T1-T4` | float (°C) / `NC` | NTC 1-4 路温度, NC=未连接 |
| `FAN` | 0/1/2 | 风扇状态 |
| `L1-L5` | 0/1 | LED 1-5 状态 |
| `P1` | `OK`/`ALARM` | 保护 1 (低压/过流) |
| `P2` | `OK`/`ALARM` | 保护 2 (高温/低温) |
| `CS` | `IDLE`/`CHARGING` | 充电状态 (CSP v1.0) |

> **注**: 字段顺序固定, 不允许插入或省略. 上位机可按 `,` 切分按位置解析, 或按 `Seq=` 这种键值对解析.

### 3.3 $EVT,CHG=... 事件推送 (CSP v1.0 新增)

单片机主动通知外部设备状态变化:

| 事件 | 触发条件 | 完整格式 |
|------|----------|----------|
| `STARTED` | IDLE → CHARGING 转换 | `$EVT,CHG=STARTED,SESSION=<id>,T=<ms>\r\n` |
| `STOPPED` | CHARGING → IDLE 转换 | `$EVT,CHG=STOPPED,SESSION=<id>,T=<ms>\r\n` |
| `HEARTBEAT_OK` | 收到 `AT+CHG=1` 但状态没变 | `$EVT,CHG=HEARTBEAT_OK,SESSION=<id>,T=<ms>\r\n` |
| `HB_TIMEOUT` | 看门狗超时未喂狗 | `$EVT,CHG=HB_TIMEOUT,SESSION=<id>,LAST_HB=<ms>,T=<ms>\r\n` |

### 3.4 bms_event_send() 二进制事件 (扩展)

通过 `bms_event_send(type, value)` 推送的低级事件, 走 #BIN_LOG 帧:

| type | value | 触发条件 |
|------|-------|----------|
| `CHG_BLOCKED` | `MOS_ON` | AT+CHG=1 时 MOS 已 ON (Rule C) |
| `MOS_BLOCKED` | `EXT_CHARGING` | AT+MOS=1 时正在充电 (Rule A) |
| `MOS_BLOCKED` | `LOW_VOLTAGE` | AT+MOS=1 时电池电压过低 |
| `MOS_BLOCKED` | `NTC_OVERHEAT` | AT+MOS=1 时 NTC 过温 |
| `MOS_BLOCKED` | `NTC_COLD` | AT+MOS=1 时 NTC 过冷 |
| `MOS_BLOCKED` | `FLIGHT_LATCH` | AT+MOS=0 时被锁存 (Rule B) |
| `RECEIVER_BINDING` | `START` / `STOPPED` | AT+BIND=1/0 触发 |
| `NTC1_OVERHEAT` ~ `NTC2_COLD` | - | NTC 临界告警 |

## 4. CSP v1.0 充电协议详解

### 4.1 AT+CHG=1 — 开始充电 / 状态确认

| 触发场景 | 外部设备检测到充电开始, 或主动确认 "我方在线" |
|---------|----------------------------------------------|
| 单片机响应 (IDLE→CHARGING) | `OK CHG=1 SRC=EXT SESSION=<id> STATE=CHARGING\r\n` |
| 单片机响应 (已在 CHARGING) | `OK CHG=1 SRC=EXT SESSION=<id> STATE=CHARGING\r\n` (SESSION 不变) |
| 事件推送 (转换) | `$EVT,CHG=STARTED,SESSION=<id>,T=<ms>\r\n` |
| 事件推送 (重复) | `$EVT,CHG=HEARTBEAT_OK,SESSION=<id>,T=<ms>\r\n` |
| 副作用 | `charge_state = CHARGING`, 启动 LED 渐进填充动画 |
|  | 若 `mosfet_enabled == 1`, 置 `mos_latched_by_chg = 1` (Rule B 锁存) |
|  | SESSION ID 自增 1 (仅 IDLE→CHARGING 转换时) |
|  | `g_chg_ext_last_seen_ms = system_time_ms` (喂狗) |
|  | 写 EEPROM 事件 `EE_EVT_CHARGE_START` |

**完整示例 (IDLE → CHARGING)**:
```
> AT+CHG=1
[CHG] External CHG=1, latching current MOS ON state    ← 仅在 MOS 已开时打印
$EVT,CHG=STARTED,SESSION=42,T=123456
< OK CHG=1 SRC=EXT SESSION=42 STATE=CHARGING
```

**完整示例 (CHARGING 期间重复, 心跳确认)**:
```
> AT+CHG=1
[CHG] External CHG=1 (heartbeat refresh)
$EVT,CHG=HEARTBEAT_OK,SESSION=42,T=128456
< OK CHG=1 SRC=EXT SESSION=42 STATE=CHARGING
```

### 4.2 AT+CHG=0 — 停止充电

| 触发场景 | 外部设备检测到充电结束 (充满 / 拔电 / 故障) |
|---------|----------------------------------------------|
| 单片机响应 | `OK CHG=0 SRC=EXT SESSION=<id> STATE=IDLE\r\n` |
| 事件推送 (CHARGING→IDLE) | `$EVT,CHG=STOPPED,SESSION=<id>,T=<ms>\r\n` |
| 副作用 | 1.5s 全亮 END_ANIM, 之后 `charge_state = IDLE` |
|  | 清 `mos_latched_by_chg = 0` (解锁 Rule B) |
|  | 清 `g_chg_external_mode = 0` (退出外部模式) |
|  | **保留** `g_chg_wdt_timeout_ms` (v3.6 改, 之前清零) |
|  | 写 EEPROM 事件 `EE_EVT_CHARGE_STOP` |
|  | SESSION ID **保留**, 不清零 |

> **注意**: 1.5s END_ANIM 期间 LED 1-4 全亮, 是正常退出动画, 之后 LED 熄灭回到 IDLE.

### 4.3 AT+CHGHB — 纯心跳 (不刷 SESSION ID)

| 触发场景 | 看门狗启用后, 外部设备周期性发心跳 |
|---------|------------------------------------|
| 单片机响应 | `OK CHGHB SESSION=<id> SRC=<EXT\|AUTO>\r\n` |
| 副作用 | 仅刷 `g_chg_ext_last_seen_ms`, 不改变 charge_state / SESSION ID |
| 事件推送 | **无** (这是与 AT+CHG=1 重复发的关键区别) |

**何时用 AT+CHGHB vs AT+CHG=1**:
- 外部设备**只关心充电状态** → 用 `AT+CHG=1` (会推 HEARTBEAT_OK, SESSION 不变)
- 外部设备**只想保活**, 不想反复确认状态 → 用 `AT+CHGHB` (更轻量)
- 默认建议: **外部设备每次只发一种**, 不要既发 CHG=1 又发 CHGHB

### 4.4 AT+CHGTO=<ms> — 看门狗超时配置

**目的**: 让外部设备**主动配置**看门狗超时, 而不是 v3.4 的硬编码 10s.

| 参数 | 范围 | 默认 (v3.6) |
|------|------|-------------|
| `<ms>` | `0` (禁用) 或 `1000..600000` (1s..600s) | `10000` (10s, 启用!) |

**单片机响应**:
- `AT+CHGTO=<ms>` (set): `OK CHGTO=<ms> ENABLED|DISABLED\r\n`
- `AT+CHGTO?` (query): `CHGTO=<ms> ENABLED|DISABLED\r\n`

**范围错误**: `ERROR: CHGTO range 0|1000..600000 ms\r\n`

**完整示例**:
```
> AT+CHGTO=15000
< OK CHGTO=15000 ENABLED

> AT+CHGTO?
< CHGTO=15000 ENABLED

> AT+CHGTO=0
< OK CHGTO=0 DISABLED
```

**触发后**:
- 启用后, 单片机在 `charge_monitor()` 检查 `(now - last_seen_ms) > g_chg_wdt_timeout_ms`
- 超时则: 推 `$EVT,CHG=HB_TIMEOUT,SESSION=<id>,LAST_HB=<ms>,T=<ms>`, 自动回退 IDLE, 清锁存
- **超时后** `g_chg_wdt_timeout_ms` **保留原值** (v3.6 改, 之前清零), 下次 `AT+CHG=1` 仍按原值生效

### 4.5 AT+CHG? — 查询状态

| 触发场景 | 上位机周期查询 (5s) 或调试 |
|---------|------------------------------|
| 单片机响应 | `+CHG=STATE,SRC=EXT\|AUTO,SESSION=<id>,HB=<ms>,STATE=...,V=...,SLOPE=...,MOS=...,MAX_V=...,DURATION=...s\r\n` |

**字段说明 (v3.5/v3.6 字段)**:

| 字段 | 取值 | 说明 |
|------|------|------|
| `+CHG=` | (前缀) | 响应标识 |
| 第 1 字段 | `IDLE` / `CHARGING` | 当前充电状态 |
| `SRC=` | `EXT` / `AUTO` | 状态来源: EXT=外部命令, AUTO=dV/dt 后备 |
| `SESSION=` | int | 当前 SESSION ID, 0=未开始过任何会话 |
| `HB=` | int (ms) | 看门狗超时配置 (0=禁用) |
| `STATE=` | `IDLE` / `CHARGING` | 状态冗余字段, 方便脚本 parse |
| `V=` | float (V) | 当前电池电压 |
| `SLOPE=` | float (V/s) | 最近 15s 电压斜率 (dV/dt 后备用) |
| `MOS=` | 0 / 1 | 当前 MOS 状态 |
| `MAX_V=` | float (V) | 本次充电会话最高电压 |
| `DURATION=` | int (s) | 已充电秒数 (IDLE 时为 0) |

**完整示例**:
```
> AT+CHG?
< +CHG=CHARGING,SRC=EXT,SESSION=42,HB=15000,STATE=CHARGING,V=12.34,SLOPE=0.0012,MOS=1,MAX_V=12.50,DURATION=120s
```

### 4.6 AT+CHGRST — 强制重置 (调试用)

| 触发场景 | 调试, 清状态机回 IDLE |
|---------|------------------------|
| 单片机响应 | `OK charge state reset to IDLE (session_id=<id> kept)\r\n` |
| 副作用 | 清充电状态、环形 buffer、END_ANIM、外部模式锁存标志 |
|  | **v3.6 改**: 复位看门狗到默认 10000ms (之前是清零) |
|  | SESSION ID **保留** (便于追溯) |

> **警告**: 此命令仅用于调试, **生产代码不要调用**!

## 5. 看门狗机制 (Watchdog Mechanism)

### 5.1 设计理念 (v3.6 默认开启)

**v3.6 关键变更**: 看门狗从"可选, 默认禁用"改为"**默认开启 10 秒超时**".

| 版本 | 开机默认 | 启用方式 | 关闭方式 |
|------|----------|----------|----------|
| v3.4 | 硬编码 10s (强制) | N/A | N/A |
| v3.5 | 禁用 (`HB=0`) | `AT+CHGTO=10000` | `AT+CHGTO=0` |
| **v3.6** | **启用 10s (`HB=10000`)** | (无需) | `AT+CHGTO=0` |

**v3.6 默认场景**:
- 开机后 `AT+CHG?` 返回 `+CHG=IDLE,SRC=AUTO,SESSION=0,HB=10000,...`
- 外部设备发 `AT+CHG=1` → 进入 CHARGING, **从此刻起 10s 内必须再喂一次狗**
- 喂狗方式: `AT+CHG=1` (会推 HEARTBEAT_OK) 或 `AT+CHGHB` (纯心跳)
- 若 10s 内没喂狗 → 单片机推 `$EVT,CHG=HB_TIMEOUT,...` + 回退 dV/dt

### 5.2 三种工作模式

#### 模式 A: 一次性命令 (低功耗)

适用: 充满后不想发任何命令的低功耗 MCU.

**v3.6 必须先发 `AT+CHGTO=0` 禁用看门狗, 否则 10s 后触发 HB_TIMEOUT**:

```c
// 启动: 禁看门狗 + 通知
bmsSerial.println("AT+CHGTO=0");
bmsSerial.println("AT+CHG=1");

// 充满 / 拔电: 通知
bmsSerial.println("AT+CHG=0");
```

> **风险**: 如果 `AT+CHG=0` 没发出去 (通信故障), 单片机永远卡在 CHARGING.
> **缓解**: 现场用 dV/dt 应急, 或用户手动 `AT+CHGRST` (复位看门狗到默认 10000U).

#### 模式 B: 3-5s 心跳 (推荐, 高可靠)

适用: 充电器主控, 4G 图传 OSD, 任何持续在线的设备.

**v3.6 无需 `AT+CHGTO`, 默认 10s 已开**:

```c
// 启动: 直接通知 (默认看门狗已开)
bmsSerial.println("AT+CHG=1");

// 循环: 每 3-5s 发一次心跳 (留 2-3x 余量给 10s 阈值)
if (millis() - last_hb > 3000) {
    last_hb = millis();
    bmsSerial.println("AT+CHGHB");  // 纯心跳, 不刷 SESSION
}

// 停止: 通知
bmsSerial.println("AT+CHG=0");
```

#### 模式 C: 状态查询为主 (上位机/调试)

```python
import serial, time
ser = serial.Serial('COM4', 115200)

ser.write(b"AT+CHG=1\r\n")

while True:
    time.sleep(5)
    ser.write(b"AT+CHG?\r\n")
    # BMS_STATUS 5Hz 自动喂狗, 通常不需要显式心跳
```

### 5.4 v3.6 升级迁移 (重要!)

**老用户 (用 v3.4/v3.5)**:
- v3.4 是硬编码 10s, 单片机自己盯着外部设备
- v3.5 是默认禁用, 用 `AT+CHGTO=10000` 启用
- v3.5 之前用"永超时"模式的用户, 升级 v3.6 后会**发现状态 10s 闪烁一次**

**典型问题场景**:
1. 升级 v3.6 固件
2. 发 `AT+CHG=1` 进入充电
3. 10s 内没喂狗 → 单片机推 HB_TIMEOUT, 回退 IDLE
4. dV/dt 接管, 若充电还在继续会重新进入 CHARGING
5. 用户体验: **充电状态闪烁 10s 一次**

**解决办法**:

| 方案 | 命令序列 | 适用场景 |
|------|----------|----------|
| A. 改用 3-5s 心跳 | `AT+CHG=1` + 每 3-5s 发 `AT+CHGHB` | **推荐**, 高可靠场景 |
| B. 显式禁用看门狗 | 升级后立刻 `AT+CHGTO=0`, 再发 `AT+CHG=1` | 低功耗一次性命令 |
| C. 改用 AT+CHGTO=60000 长超时 | `AT+CHGTO=60000` + `AT+CHG=1` + 每 30s 心跳 | 不在意偶尔失联 |

## 6. 安全规则 (Safety Rules)

### 6.1 Rule A: 充电闭锁 (Charging Lockout)

**目的**: 防止地面充电时 (CHARGING 状态) 误开 MOS, 导致电池带载充电.

**条件**: `g_chg_external_mode == 1` AND `charge_state == CHG_STATE_CHARGING`

**行为**:
- `mosfet_control(1)` 返回 0, 拒绝开启
- 播放失败动画 (3 次 LED 闪烁)
- 发送 `bms_event_send("MOS_BLOCKED", "EXT_CHARGING")` 二进制事件

**单片机日志**:
```
[MOS] Blocked - external reports CHARGING, refuse to enable
```

> **注**: dV/dt 后备模式 (AUTO) 不应用 Rule A.

### 6.2 Rule B: 飞行锁存 (Flight Latch)

**目的**: 防止飞行中充电器意外插入后, 任何外部干扰切断正在供电的 MOS (= 坠机).

**条件**: `mos_latched_by_chg == 1`

**行为**:
- `mosfet_control(0)` 返回 0, 拒绝关闭 (AT/上位机路径)
- 发送 `bms_event_send("MOS_BLOCKED", "FLIGHT_LATCH")` 二进制事件

**单片机日志**:
```
[MOS] Blocked - latched ON by CHARGING, cannot turn off (use key for emergency)
```

**豁免 (Key Path)**:
- **按键路径** (主循环按键扫描) 可清除锁存标志后强制 OFF
- 这是**飞行员紧急切断逃生口**, 不受 Rule B 保护
- 触发后单片机日志: `[Key] Clear flight latch (MOS OFF by user)`

**清锁存**:
- `AT+CHG=0` 外部命令
- `AT+CHGRST` 调试命令
- 心跳超时回退
- 按键 OFF 路径

### 6.3 Rule C: MOS ON 禁充 (MOS Load-Active Lockout) — v3.6 新增

**目的**: 防止 MOS ON (负载激活) 时启动充电, 避免电池既充又放.

**条件**: `mosfet_enabled == 1`

**行为**:
- `AT+CHG=1` 返回 `BLOCKED CHG=1 REASON=MOS_ON`, 不进入外部模式
- 发送 `bms_event_send("CHG_BLOCKED", "MOS_ON")` 二进制事件
- 提示信息: `[CHG] Blocked - MOS is ON (load active), cannot start charging`

**豁免** (以下命令**不受** MOS 状态影响):
- `AT+CHG=0`
- `AT+CHGHB`
- `AT+CHGTO=<ms>`
- `AT+CHG?`
- `AT+CHGRST`

**单片机日志**:
```
> AT+CHG=1
[CHG] Blocked - MOS is ON (load active), cannot start charging
  hint: turn off MOS first, then retry AT+CHG=1
< BLOCKED CHG=1 REASON=MOS_ON
```

**与其他规则的关系**:
- Rule A 是 "充电中不开 MOS" (CHARGING → 拒绝 mosfet_control(1))
- Rule C 是 "MOS ON 中不充" (mosfet_enabled=1 → 拒绝 AT+CHG=1)
- **Rule A 与 Rule C 互为对称**, 共同防止"电池既充又放"的危险状态

# 第二部分: Web BMS 协议 (v3.18 WSP v3.18)

## 7. Web BMS v3.18 架构

### 7.1 三角色模型 (Three-Role Model)

v3.18 引入 **三角色彻底解耦** 架构:

| 角色 | 含义 | 唯一性 | 触发 |
|------|------|--------|------|
| **WEB_SERIAL_OWNER** | 谁开了物理 WebSerial (= 数据源 PC) | 全局唯一 | 自己点 "选择并连接串口" 选 COM 口 |
| **ACTIVE_CONTROLLER** | 谁有"抢控"语义 (v1.4 强抢, 向后兼容) | 全局唯一 | `claim_control` 消息 |
| **DOMINANT** | 谁能发 AT 命令 (v3.15 引入, 需握手) | 全局唯一 | `request_dominant` + 批准 |

**关键解耦**: 三个角色**可以是三台不同的 PC**:
- PC-A 开了 WebSerial → 是 WEB_SERIAL_OWNER (数据源, 被动广播)
- PC-B 抢了 controller → 是 ACTIVE_CONTROLLER (v1.4 兼容, 实际不直接用)
- PC-C 申请并被批准 DOMINANT → 唯一能发 AT (经后端透传给 PC-A 的 WebSerial)

### 7.2 整体架构图

```

            ┌──────────────────────┼──────────────────────┐
```

### 8.2 客户端 → 服务端消息

| type | 必填字段 | 用途 | 权限 |
|------|----------|------|------|
| `claim_control` | (无) | 抢 / 接管控制权 (v1.4 允许强抢) | 任何客户端 |
| `release_control` | (无) | 主动释放控制权 | 仅 ACTIVE_CONTROLLER |
| `request_dominant` | (无) | 申请成为主导 (发 AT 权限) | 任何客户端 |
| `approve_dominant` | `target_cid` | 批准别人的主导申请 | 仅当前 DOMINANT |
| `deny_dominant` | `target_cid` | 拒绝别人的主导申请 | 仅当前 DOMINANT |
| `release_dominant` | (无) | 主动释放主导权 | 仅当前 DOMINANT |
| `serial_data` | `data` (string) | 上报从串口读到的 1 行 | 任何客户端 (v3.18 移除门控) |
| `serial_status` | `connected`, `port?`, `baud?` | 上报 WebSerial 状态变化 | 任何客户端 (v3.18 移除门控) |
| `subscribe` | `master_id` ("..." or "*") | 显式订阅 (默认 * 已自动) | 任何客户端 |
| `unsubscribe` | (无) | 取消订阅 (变静默) | 任何客户端 |
| `set_broadcast` | `enabled` (bool) | 关闭 / 开启透传 (默认开) | 任何客户端 |
| `cmd` | `data` (string) | 远程 AT 命令 (后端转发给 owner) | **仅 DOMINANT** |

| type | 字段 | 触发时机 |
|------|------|----------|

| `cmd` | `data`, `from` (cid) | 远程 AT 命令 → 透传给 WEB_SERIAL_OWNER |

### 11.1 Arduino 充电器主控 (模式 B: 30s 看门狗)

```c
#include <SoftwareSerial.h>
SoftwareSerial bmsSerial(2, 3);  // RX, TX

uint32_t last_heartbeat_ms = 0;
const uint32_t HEARTBEAT_INTERVAL = 10000;  // 10s 心跳 (3x 余量给 30s 超时)
bool was_charging = false;
uint32_t current_session_id = 0;

void parse_evt(const String& line) {
    if (!line.startsWith("$EVT,CHG=")) return;
    int eq1 = line.indexOf('=', 9);
    String evt = line.substring(9, eq1);
    if (evt == "HB_TIMEOUT") {
        Serial.println("ALARM: BMS watchdog timeout!");
    }
}

void loop() {
    bool is_charging = digitalRead(CHARGE_DETECT_PIN) == HIGH;
    float voltage = analogRead(VOLTAGE_SENSE_PIN) * 0.00488;

    while (bmsSerial.available()) {
        String line = bmsSerial.readStringUntil('\n');
        parse_evt(line);
    }

    if (is_charging && !was_charging) {
        bmsSerial.println("AT+CHGTO=30000");
        bmsSerial.println("AT+CHG=1");
        last_heartbeat_ms = millis();
    } else if (!is_charging && was_charging) {
        bmsSerial.println("AT+CHG=0");
        bmsSerial.println("AT+CHGTO=0");
    } else if (is_charging) {
        if (millis() - last_heartbeat_ms > HEARTBEAT_INTERVAL) {
            last_heartbeat_ms = millis();
            bmsSerial.println("AT+CHGHB");
        }
    }
    was_charging = is_charging;
    delay(100);
}
```

### 11.2 Python 上位机 (CSP 调试工具)

```python
import serial, time

def send_chg_with_retry(ser, val: int, max_retries=3, timeout_ms=500):
    """带 3 次重试的 AT+CHG={val}, v3.6 推荐模式"""
    cmd = f"AT+CHG={val}\r\n"
    expect_prefixes = [f"OK CHG={val}", f"BLOCKED CHG={val}"]
    for attempt in range(1, max_retries + 1):
        ser.reset_input_buffer()
        ser.write(cmd.encode())
        end = time.time() + timeout_ms / 1000.0
        buf = b""
        while time.time() < end:
            n = ser.in_waiting
            if n:
                buf += ser.read(n)
                for line in buf.decode('utf-8', errors='replace').split('\n'):
                    line = line.strip()
                    for pfx in expect_prefixes:
                        if line.startswith(pfx):
                            return line
            time.sleep(0.02)
        print(f"[retry {attempt}/{max_retries}] timeout, retry in 100ms")
        time.sleep(0.1)
    raise TimeoutError(f"AT+CHG={val} 失败, 设备可能离线")


# 模式 B: 3-5s 心跳 (推荐)
ser = serial.Serial('COM4', 115200)
print(send_chg_with_retry(ser, 1))  # 开始充电

try:
    while True:
        time.sleep(3)
        ser.write(b"AT+CHGHB\r\n")  # 纯心跳, 不刷 SESSION
except KeyboardInterrupt:
    print(send_chg_with_retry(ser, 0))  # 停止充电
    ser.close()
```

### 11.3 Web 端 (浏览器 JavaScript) — 远程 AT 透传

```javascript
// 远程 PC 申请主导 (作为示例, 实际用页面 UI)
const ws = new WebSocket(`ws://${location.host}/ws`);

ws.onopen = () => {
    // 1. 申请主导 (会弹申请给 PC-甲 或现任主导)
    ws.send(JSON.stringify({ type: "request_dominant" }));

ws.onmessage = (e) => {
    const msg = JSON.parse(e.data);

    if (msg.type === "dominant_status" && msg.active === myClientId) {
        // 我被批准了, 现在可以发 AT
        console.log("我是主导了, 发 AT:");
        ws.send(JSON.stringify({
            type: "cmd",
            data: "AT+MOS?\r\n"
        }));
    }

    if (msg.type === "cmd" && msg.from) {
        // 我是数据源 PC, 收到远程 AT, 写入 WebSerial
        navigator.serial.getPorts().then(ports => {
            const port = ports[0];
            port.open({ baudRate: 115200 }).then(() => {
                const writer = port.writable.getWriter();
                writer.write(new TextEncoder().encode(msg.data));
                writer.releaseLock();
            });
        });
    }

    if (msg.type === "error") {
        console.error("后端错误:", msg.reason);
    }
};
```

## 12. 典型场景 (Typical Scenarios)

### 12.1 场景 A: 单 PC 本地调试 (基础场景)

**硬件**: 1 台 PC, USB 接 STM32.

**用户操作**:
1. 启动后端: `python bms_web_backend.py`
2. 浏览器开 `http://localhost:5091/`
3. 点 "选择并连接串口" → 选 COM4 → 连接
4. 点 "我要调试" (或申请主导)
5. 在 AT 输入框输 `AT+MOS?` → 收到响应

**后端状态** (1 客户端, 三角色都可能是自己):
- `clients`: `["A3B9F2"]`
- `web_serial_owner`: `A3B9F2` (自己)
- `dominant`: `A3B9F2` (自己)

### 12.2 场景 B: 双 PC 数据源 + 远程调试 (推荐多 PC 配置)

**硬件**:
- PC-甲 (192.168.2.44): 插着 STM32, 作为数据源 PC
- PC-乙 (192.168.2.X): 远程 PC, 想调试 BMS

**部署**:
1. PC-甲 上跑后端, 开放防火墙 5091
2. PC-甲 浏览器开 `http://localhost:5091/`, 开 WebSerial (成为 WEB_SERIAL_OWNER)
3. PC-乙 浏览器开 `http://192.168.2.44:5091/`, 看到数据源是 PC-甲
4. PC-乙 点 `🚀 申请主导` → PC-甲 收到同意/拒绝弹窗 → PC-甲 同意
5. PC-乙 现在是 DOMINANT, 可发 AT
6. PC-乙 发 `AT+MOS?` → 后端透传给 PC-甲 → PC-甲 写入 WebSerial → MCU 响应 → 透传回 PC-乙

**后端状态** (2 客户端, 三角色解耦):
- `clients`: `["A3B9F2 (PC-甲)", "B7E1D4 (PC-乙)"]`
- `web_serial_owner`: `A3B9F2` (PC-甲)
- `dominant`: `B7E1D4` (PC-乙, 申请被批准)
- `active`: `null` 或 `A3B9F2` (v1.4 兼容, 通常无意义)

### 12.3 场景 C: 多人协作 (1 数据源 + 2 远程)

**硬件**:
- PC-甲: 数据源 (开 WebSerial, 不申请主导)
- PC-乙: 远程观察员 (只看不发)
- PC-丙: 远程调试员 (申请主导, 发 AT)

**用户操作**:
- PC-甲: 启动后端 + 开 WebSerial
- PC-乙: 浏览器开 → 只看数据
- PC-丙: 浏览器开 → 申请主导 (PC-甲 批准) → 发 AT

**注意**: **同一时刻只能有一个 DOMINANT** (v3.15 强制). PC-丙 在任时, PC-乙 即使点申请也会被拒.

### 12.4 场景 D: 数据源切换 (谁开 WebSerial 谁就是 owner)

1. PC-甲 已开 WebSerial (是 WEB_SERIAL_OWNER)
2. PC-丙 也开 WebSerial → 后端自动让 PC-丙 成为新 owner, PC-甲 变普通客户端
3. 后端日志: `[PC-丙] 替换 WebSerial owner (原=PC-甲)`
4. PC-甲 收到的数据从 "自己读自己" 变成 "PC-丙 透传过来"

> **注意**: 数据源切换是**静默发生**的, 不会弹窗提示. 若需多人数据源, 请用 v1.4 旧的 claim_control 模型 (但 v3.18 不再推荐).

## 13. 故障排查 (Troubleshooting)

### 13.1 浏览器连不上 (页面白屏/连接失败)

| 现象 | 原因 | 解决 |
|------|------|------|
| 本机 `http://localhost:5091/` 也打不开 | 后端没启动 | `python bms_web_backend.py` |
| 远程 `http://192.168.x.x:5091/` 打不开, 本机能开 | 防火墙挡了 | 跑 `setup_firewall.bat` (管理员) |
| 远程用了 `http://localhost:5091/` | localhost 永远是"自己 PC" | 改成 `http://PC-甲的真实IP:5091/` |
| 远程 IP 老是变 | PC-甲 用 DHCP | 给 PC-甲 设静态 IP, 或开 WiFi 热点 (192.168.137.1 固定) |
| 用手机/平板访问连不上 | 跨网段 / 公网 | 主机防火墙开 5091, 或用 frp 公网反代 |

### 13.2 ssw 显示 "📡 数据源: 无"

| 排查项 | 检查方法 |
|--------|----------|
| PC-甲 浏览器开 WebSerial 了? | PC-甲 浏览器点 "选择并连接串口" → 选 COM4 → 波特率 115200 |
| WebSerial 实际连上了? | PC-甲 浏览器 console (F12) 应该有 `WebSerial connected` |
| COM4 被别的程序占? | 关闭 PyQt5 上位机 / 串口助手 / 其他浏览器 tab |
| PC-甲 浏览器在哪个 URL? | 必须是 `http://localhost:5091/` 或 `http://192.168.2.44:5091/`, **不能**用 `http://192.168.2.X:5091/` (PC-甲 自己 IP 之外的其他 IP) |
| 防火墙挡了 WebSocket? | 防火墙只挡 5091 TCP, 不区分 HTTP/WS, 开了就行 |

### 13.3 远程 PC 看不到数据帧

**先确认 PC-甲 自己的浏览器能看到数据 (能滚 `$BMS_STATUS`)**:
- ✓ 能: PC-甲 没问题, 是后端没广播
- ✗ 不能: PC-甲 都没数据, 远程更不会有

**若 PC-甲 能, 远程不能**:

| 排查项 | 方法 |
|--------|------|
| 远程 ssw 显示 "📡 数据源: PC-甲" ? | 是 → 后端有透传, 检查远程浏览器是否卡死/缓存 (`Ctrl+Shift+R`) |
| 远程 ssw 显示 "📡 数据源: 无" ? | PC-甲 没上报 serial_status, 回到 §13.2 |
| 远程 F12 console 有红色报错? | 把错误截图发我 |
| 远程浏览器 URL 是不是 `http://192.168.2.44:5091/`? | 不能用 `localhost` (那是它自己) |
| 后端日志看 PC-甲 有没有发 serial_data? | 看任务 `bj07t0n1w` 输出 |

### 13.4 远程发 AT 没反应

| 现象 | 原因 | 解决 |
|------|------|------|
| AT 输入框灰色, 提示 "需先申请主导" | 不是 DOMINANT | 点 `🚀 申请主导` |
| 申请被 PC-甲 弹窗拒绝 | PC-甲 选了"拒绝" | 重申请 + 通知 PC-甲 同意 |
| 申请后等 10s 无回应 | 现任主导离线 / 没收到 | 现任主导 30s 后自动释放, 重申请 |
| AT 发送后 Toast "当前无数据源 PC" | WEB_SERIAL_OWNER 离线 | 让数据源 PC 重新开 WebSerial |
| AT 发送后 3s 无响应 | 网络/串口问题 | 检查后端 log + WebSerial 状态 |

### 13.5 端口冲突 (重要!)

**症状**: 启动后端报 `Address already in use` 或多次启动多个后端.

**排查**:
```cmd
netstat -ano | findstr "5091"
# 或
netstat -ano | findstr "5000"
```

**清理**:
```cmd
# 看 PID 后
taskkill /F /PID <pid>
```

**根因**: 之前误启了多个后端, 或 `start_helper.py` 默认端口是 5000, 跟现在用的 5091 不一致.

**解决**: 修改 `start_helper.py` 默认端口到 5091, 或启动时显式 `--http 5091`.

### 13.6 WebSerial 弹窗没反应 / 拒绝授权

| 现象 | 原因 | 解决 |
|------|------|------|
| 点 "选择并连接串口" 完全没反应 | 浏览器不支持 WebSerial | 用 Chrome 89+ / Edge 89+ |
| 弹端口框后选完没打开 | 权限被拒绝 / 端口被占 | 重新刷新页面, 重试 |
| 之前授权过但这次报错 | Chrome 缓存站点权限异常 | `chrome://settings/content/serialPorts` 删除授权 |
| 在非 secure context 下用 | `http://192.168.x.x` 在某些浏览器不算 secure | 改用 `http://localhost` (本机) 或部署 HTTPS |

## 14. 兼容性 (Compatibility)

### 14.1 与旧版 Web BMS 兼容

| 旧版 | 兼容情况 | 升级建议 |
|------|----------|----------|
| v1.0 (后端独占 COM) | ✗ 协议不兼容, 但物理可共存 (同时只一个连 USB) | 用 v3.18 替换 |
| v1.1 (WebSerial + 强抢 controller) | ✓ 协议层兼容 (client_list / serial_data 字段都在) | 推荐升级到 v3.18, 享受数据源 PC 解耦 |
| v3.15 (DOMINANT 模型) | ✓ DOMINANT 相关协议不变 | 数据源角色解耦 (v3.16+) 是新功能 |
| v3.16 (数据源 PC) | ✓ serial_status 上报规则不变 | v3.18 修复了透传门控 bug |
| v3.17 (本地 owner 短路) | ✓ 客户端行为 | 修了一些列外场景 |

### 14.2 与 PyQt5 上位机 (BMS_PC_Software_v2) 共存

- **同一物理 USB 不能同时被两个进程独占**: PyQt5 占 USB 时, 浏览器 WebSerial 打开会失败 (反之亦然)
- **协议层完全兼容**: 都走 AT 命令 + 5Hz `$BMS_STATUS`, 上位机可同时存在, 看同一个 BMS
- **建议分工**: 现场调试员用 PyQt5 (功能最全), 远程观察员用 Web (零安装)

### 14.3 $BMS_STATUS 字段兼容性

字段格式**不变**, 仅多了 `CS=` 字段 (IDLE/CHARGING). 旧上位机 regex 若用 `,` split 然后按位置取, 需更新位置索引; 用键值对解析的 (`Seq=`, `VW=`...) 不受影响.

## 15. WebSerial 浏览器兼容性

### 15.1 浏览器支持

| 浏览器 | WebSerial | 备注 |
|--------|-----------|------|

| `https://域名` | ✓ 需 TLS 证书 |

**生产环境**: 用 frp / nginx / caddy 反代 + Let's Encrypt 证书, 走 HTTPS.

2. 选中要授权的设备

**清除授权**: `chrome://settings/content/serialPorts` 删除站点数据.

### A.4 $BMS_STATUS 字段表


| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|

| `CS` | str | IDLE/CHARGING | 充电状态 (CSP v1.0) |

## 附录 C: 故障处理矩阵 (v3.6 + v3.18 综合)

| 故障场景 | v3.6 固件行为 | v3.18 Web 行为 |
|---------|--------------|----------------|
| 外部设备发 `AT+CHG=1` 后断开 (无看门狗) | 10s 后推 `$EVT,CHG=HB_TIMEOUT` + 回退 dV/dt | (无影响, UART 协议层) |
| `AT+CHG=1` 期间用户双击按键开 MOS | 允许 (按键豁免 Rule A), 触发 Rule B 锁存 | (无影响) |
| 锁存后上位机发 `AT+MOS=0` | 拒绝, `[MOS] Blocked - FLIGHT_LATCH` | (无影响) |
| 锁存后外部设备发 `AT+CHG=0` | 解锁, END_ANIM, MOS 可关 | (无影响) |
| MOS ON 时外部设备发 `AT+CHG=1` | `BLOCKED CHG=1 REASON=MOS_ON` | (无影响) |
| **Web: PC-甲 数据源, 关闭浏览器** | (无影响) | 后端 30s 内自动释放 web_serial_owner, 远端 ssw 变 "📡 数据源: 无" |
| **Web: 主导 PC 关闭浏览器** | (无影响) | 后端 30s 宽限后自动释放 dominant, 远端 ssw 变 "⏸ 无主导" |
| **Web: PC-甲 + PC-乙 同时开 WebSerial** | (无影响, WebSerial 各自独立) | 后端让后开的为 owner, 日志记录替换 |
| **Web: 超过 3 个客户端连** | (无影响) | 第 4 个 WebSocket 升级失败, 浏览器显示 "已达 max_clients=3" |
