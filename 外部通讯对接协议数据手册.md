## 1. 概述 (Overview)

### 1.1 文档范围

本手册覆盖 **AMKN8639 机库控制板对外的两条通讯链路**:

```
┌─────────────────────────────────────────────────────────────┐
│              AMKN8639 机库控制板 (STM32H743)                   │
│                                                             │
│   UART1 (GSP v1.0, RS232 115200 8N1)                       │
│     ↑↓ AT+OPEN/CLOSE/...  $GARAGE,...  $EVT,...  $FAULT    │
│                                                             │
│   USART2 / SWD (内部调试用, 不在本手册范围)                  │
└─────────────────────────────────────────────────────────────┘
       ↑↓                                  ↑↓
   [RK3568 上位机]                       [ROS2 节点]
   (Ubuntu + hangar_bridge)               (任意多台)
       ↓                                      ↓
   ROS2 topics: /hangar/state, /hangar/fault, /hangar/cmd
```

**关键事实**: 上位机与机库之间**只有一条物理串口 UART1**, RK3568 上的 `hangar_bridge` 节点独占访问, 其他 ROS2 节点通过 ROS2 topic 与 bridge 通信.

| 读者 | 看哪几章 |
|------|----------|
| **ROS2 节点开发者** | 第 2-5 章 (UART 物理层/协议/保护规则), 第 11 章集成示例 |
| **机库二次集成工程师** | 第 6-9 章 (架构/消息契约/调试), 第 11 章 |
| **现场调试员** | 第 1, 10, 12-15 章 (部署, 故障排查, 兼容性) |

# 第一部分: UART 协议 (固件 V1.31.02 GSP v1.0)

## 2. 物理层 (Physical Layer)

| 项目 | 规格 |
|------|------|
| 串口 | USART1 (TTL 3.3V / RS232) |
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 校验位 | None |
| 停止位 | 1 |
| 流控 | None |
| 帧格式 | ASCII 文本, `\r\n` 结尾 |
| 电平 | 3.3V TTL 或 RS232 (板载 SP3232 转换) |
| 字符集 | ASCII 7-bit (命令字段) + GBK (中文 $EVT 字段) |
| 最大行长 | 128 字节 (MCU AT_PushChar 限制) |

### 2.1 引脚定义

| 单片机引脚 | 功能 | 外部设备连接 |
|------------|------|--------------|
| PA9 (USART1_TX) | 单片机发送 | 接 RK3568 RX |
| PA10 (USART1_RX) | 单片机接收 | 接 RK3568 TX |
| GND | 共地 | **必须** 共地 |

### 2.2 隔离建议

工业环境强烈建议加 **数字隔离器** (ADuM1411 / ISO1050):
- 隔离器供电: 单片机侧 3.3V, 外部设备侧独立供电
- 隔离耐压: ≥ 2.5kV (基本) / ≥ 4kV (推荐)
- **注意**: AMKN8639 板载 SP3232 已做 TTL↔RS232 转换, 但与 RK3568 直连时**不需要** RS232, 直接用 3.3V TTL 即可.

## 3. 协议 (Protocol)

外部设备通过 AT 命令集 + 事件推送与机库单片机通信. 所有命令以 `\r\n` 结尾.

### 3.1 命令总览 (按功能分组)

#### 机械控制 (5 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+OPEN` | 双向 | 打开舱盖 (X 电机正转) |
| `AT+CLOSE` | 双向 | 关闭舱盖 (X 电机反转) |
| `AT+CENTER` | 双向 | 归中 (Y 电机正转, 推板向中心) |
| `AT+RELEASE` | 双向 | 释放 (Y 电机反转, 推板向外) |
| `AT+STOP` | 双向 | 立即停电机 (不动 K2) |

#### 充电控制 (4 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+CHGENA` | 双向 | **手动**: 强制 K2/K3/K4 接通, 禁用自动判定 |
| `AT+CHGDIS` | 双向 | **手动**: 强制 K2/K3/K4 断开, 禁用自动判定 |
| `AT+CONSERVATIVE` | 双向 | 进入 P7 保守模式 (停电机 + 断 K2) |
| `AT+RESUME` | 双向 | 退出 P7 保守模式, 回到 IDLE_CLOSED |

#### 链路保护策略 (2 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+RKPROT=0` | 双向 | P7 链路丢失**仅告警** (默认) |
| `AT+RKPROT=1` | 双向 | P7 链路丢失**即进入保守模式** |
| `AT+BMSPROT=0` | 双向 | BMS 离线**仅告警** (默认) |
| `AT+BMSPROT=1` | 双向 | BMS 离线**即脱扣 K2** |

#### 系统 / 诊断 (10 条)

| 命令 | 方向 | 用途 |
|------|------|------|
| `AT+HELP` | 双向 | 显示所有命令 |
| `AT+VER` | 双向 | 查询固件版本 |
| `AT+RESET` | 双向 | 软复位 MCU |
| `AT+HB` | 双向 | 心跳 (无副作用, 仅刷新 RX 时间戳) |
| `AT+DI?` | 双向 | 读 DI1..8 |
| `AT+AI?` | 双向 | 读 8 路 AI 原始值 (12-bit) |
| `AT+DO=<n>,<v>` | 双向 | 直写 DO (n=1..16, v=0/1) |
| `AT+AO1=<v>` / `AT+AO2=<v>` | 双向 | DAC1/DAC2 输出 (0..4095) |
| `AT+RTC?` | 双向 | 提示使用 `AT+RTC=?` (libapp) |
| `AT+FAULTLOG?` | 双向 | 读 EEPROM 故障日志 (32 条) |
| `AT+FAULTLOG=CLEAR` | 双向 | 清 EEPROM 故障日志 |
| `AT+SELFTEST` | 双向 | 启动 9 步故障注入 |
| `AT+SELFTEST=ABORT` | 双向 | 中止故障注入 |
| `AT+BMSPOWER=ON` / `OFF` / `ABORT` | 双向 | K5 短-短-长按键序列 |
| `AT+ALARM=OFF` | 双向 | **新增**: 立即关蜂鸣器 |
| `AT+OTA` | 双向 | 双 Bank OTA 升级 |
| `AT+FWU` | 双向 | YModem 固件升级 |

### 3.2 $GARAGE 主动帧 (2Hz)

单片机每 500ms 主动推一帧状态:

```
$GARAGE,State=IDLE_CLOSED,Cover=CLOSED,Plates=CC,Dock=OFF,BMS=NO_LINK,SOC=0,V=0,I=0,Conservative=NO
```

| 字段 | 取值 | 说明 |
|------|------|------|
| `State` | `IDLE_CLOSED`/`OPENING`/`IDLE_OPEN`/`CENTERING`/`PLATES_CENTER`/`RELEASING`/`CLOSING` | 7 状态机当前态 |
| `Cover` | `CLOSED`/`OPEN`/`MOVING` | 舱盖位置 (DI2 派生) |
| `Plates` | 2 ASCII 字符, e.g. `CC`, `OO`, `MM`, `MO` | A 板+B 板, `C`=中位 `O`=外位 `M`=运动中 |
| `Dock` | `ON`/`OFF` | K2/K3/K4 接触器状态 |
| `BMS` | `LINK`/`NO_LINK` | 飞机 BMS 链路 |
| `SOC` | 0..100 (%) | 飞机 BMS 剩余容量 |
| `V` | 0..4095 (12-bit ADC) | AI1 原始值 (充电电压) |
| `I` | 0..4095 (12-bit ADC) | AI2 原始值 (充电电流) |
| `Conservative` | `YES`/`NO` | P7 保守模式标志 |

> **注**: 字段顺序固定, 不允许插入或省略. ROS2 节点按 `,` 切分按位置解析, 或按 `State=` 这种键值对解析.
> **注 2**: `V`/`I` 仅在 `State=PLATES_CENTER && Dock=ON` 时刷新, 其他状态是上次的 stale 值.

### 3.3 $EVT,<NAME>,<detail> 事件推送 (5Hz 节流)

> 节流: 200ms 最小间隔. 突发事件被丢弃而非排队.

| `NAME` | `detail` | 触发条件 | 中文释义 |
|------|----------|----------|----------|
| `STATE` | `GOTO=<STATE_NAME>` | 每次状态机迁移 | 状态变更 |
| `STOP` | (无) | `AT+STOP` 收到 | 主机命令停电机 |
| `RESUMED` | (无) | `AT+RESUME` 收到 | 退出 P7 |
| `CONSERVATIVE` | `RK_LOST` 或 `CMD` | P7 模式进入 | 进入保守模式 |
| `RK_LOST` | `IGNORED` | 10s UART 静默 (默认) | RK 链路丢失 (仅告警) |
| `RK_RESTORED` | (无) | 链路恢复 (重放 lastRkCmd) | RK 链路恢复 |
| `DOCK=ON` | `MANUAL` 或 `AUTO` | `AT+CHGENA` 或自动判定 | K2 接通 |
| `DOCK=OFF` | `MANUAL`/`AUTO_FULL`/`PRE_TAKEOFF`/`BMS_OFFLINE` | `AT+CHGDIS`/充满/起飞前/BMS 离线 | K2 断开 |
| `BMS_OFFLINE` | `IGNORED` | BMS 离线且默认不脱扣 | BMS 离线告警 |
| `EMG=PRESSED` | (无) | 急停按下 (DI1=0) | 急停按下 |
| `EMG=RELEASED` | (无) | 急停松开 3s 后 | 急停释放 |
| `COVER_OPEN_TIMEOUT` | (无) | P1: 舱盖开 >30s | 开超时 |
| `COVER_CLOSE_TIMEOUT` | (无) | P1: 舱盖关 >30s | 关超时 |
| `CENTER_TIMEOUT` | (无) | P1: 归中 >15s | 归中超时 |
| `RELEASE_TIMEOUT` | (无) | P1: 释放 >15s | 释放超时 |
| `LIMIT_INVALID` | `OPENING_but_CLOSED`/`CLOSING_but_OPEN`/`CENTERING_at_OUTER`/`RELEASE_at_CENTER` | P2: 限位异常 | 限位异常 |
| `OVER_VOLT` | `V=high` | P4: V>16.8V | 充电过压 |
| `OVER_CURR` | `I=high` | P5: I>18mA ≥1s | 充电过流 |
| `FULL_VOLT_NO_DROP` | `V,I` | P6: V≥16.5V 且 I≥16mA ≥5s | 满电未降流 |
| `SELFTEST` | `BEGIN`/`STEP=P1..P9`/`DONE`/`ABORT` | `AT+SELFTEST` 9 步注入 | 自检 |
| `BMSPOWER` | `BEGIN`/`DONE`/`FORCE_OFF`/`ABORT` | K5 序列 | BMS 上电序列 |
| `RKPROT` | `ON`/`OFF` | `AT+RKPROT=0/1` | P7 策略切换 |
| `BMSPROT` | `ON`/`OFF` | `AT+BMSPROT=0/1` | BMS 离线策略 |
| `OTA=START`/`FAIL`/`NO_FIRMWARE`/`IAP_DISABLED` | (无) | `AT+OTA` 结果 | OTA 反馈 |
| `FWU=ENTER_YMODEM`/`YMODEM_DISABLED` | (无) | `AT+FWU` 结果 | YModem |
| `FAULTLOG` | `DUMPED`/`CLEARED` | `AT+FAULTLOG?`/`=CLEAR` 完成 | 故障日志 |
| `VERSION` | `AMKN8639 V1.31.02` | `AT+VER` | 固件版本 |
| `HELP` | 命令列表 | `AT+HELP` | 命令列表 |
| `AT` | `DI=8,NN`/`AI=8,v0..v7`/`DO=n,v`/`AO1=v`/`AO2=v` | `AT+DI?`/`AT+AI?`/`AT+DO=`/`AT+AO1=`/`AT+AO2=` | 通用 IO |
| `RTC` | `USE AT+RTC=?` | `AT+RTC?` | RTC 提示 |

### 3.4 $FAULT 故障日志

仅在收到 `AT+FAULTLOG?` 时输出, 一次最多 32 条 EEPROM 记录:

```
$FAULT,IDX=0,CODE=0x10,VAL=3520,TS=123456
```

| 字段 | 取值 | 说明 |
|------|------|------|
| `IDX` | 0..31 | EEPROM 环形索引 |
| `CODE` | 故障码 (见 §6.1 故障码表) | 故障类型 |
| `VAL` | 原始值 | 电压/电流 raw, 或超时 ms |
| `TS` | uint32 (ms) | MCU tick 时间戳 |

## 4. AT 命令详解

### 4.1 AT+OPEN — 打开舱盖

| 触发场景 | 上位机命令 "准备接收无人机" |
|---------|------------------------------|
| 单片机响应 | 触发 `STATE,GOTO=OPENING` |
| DO 输出 | DO1=1, DO2=0 (X 电机正转) |
| 退出条件 | `coverAtOpen==1` → IDLE_OPEN; 或 timeout 30s → COVER_OPEN_TIMEOUT |
| 记录 | `lastRkCmd = LAST_CMD_OPEN` (P7 链路恢复时重放) |

**完整示例**:
```
> AT+OPEN
< $EVT,STATE,GOTO=OPENING
< $GARAGE,State=OPENING,...
  (X 电机运行)
< $EVT,STATE,GOTO=IDLE_OPEN
< $GARAGE,State=IDLE_OPEN,...
```

### 4.2 AT+CLOSE — 关闭舱盖

| 触发场景 | 上位机命令 "无人机已离开" |
|---------|--------------------------|
| 单片机响应 | 触发 `STATE,GOTO=CLOSING` |
| DO 输出 | DO1=0, DO2=1 (X 电机反转) |
| 退出条件 | `coverAtClosed==1` → IDLE_CLOSED; 或 timeout 30s |

### 4.3 AT+CENTER — 归中 (推板向中心)

| 触发场景 | 上位机命令 "无人机进入中心位置" |
|---------|------------------------------|
| 单片机响应 | 触发 `STATE,GOTO=CENTERING` |
| DO 输出 | DO3=1, DO4=0 (Y 电机正转) |
| 退出条件 | 两板均到中心 → PLATES_CENTER; 或 timeout 15s |
| 副作用 | 进入 PLATES_CENTER 时**清所有充电覆盖标志**, 重启自动判定 (3s 周期) |

### 4.4 AT+RELEASE — 释放 (推板向外)

| 触发场景 | 上位机命令 "准备起飞" |
|---------|----------------------|
| 单片机响应 | 触发 `STATE,GOTO=RELEASING`, **K2 立即断** (`DOCK=OFF,PRE_TAKEOFF`) |
| DO 输出 | DO3=0, DO4=1 (Y 电机反转) |
| 退出条件 | 两板均到外侧 → IDLE_OPEN; 或 timeout 15s |

### 4.5 AT+CHGENA — 强制 K2 接通 (手动覆盖)

| 触发场景 | 上位机命令 "我方在线, 启动充电" |
|---------|-------------------------------|
| 单片机响应 | `DOCK=ON,MANUAL` 事件 |
| 副作用 | K2/K3/K4 立即接通; `chargeHostManual=1` |
| 自动判定 | **被禁用** (`ChargeAutoDecide` 早返回) |
| 保护规则 | P4/P5/P6 仍生效, 触发时仍会脱扣 K2 |
| 记录 | `lastRkCmd = LAST_CMD_CHGENA` (P7 重放) |

> ⚠️ **手动不能关闭保护**: 即使 `AT+CHGENA`,P4/P5/P6 触发仍会立即 `DropCharge()` 并返回.

### 4.6 AT+CHGDIS — 强制 K2 断开 (手动覆盖)

| 触发场景 | 上位机命令 "停止充电" |
|---------|----------------------|
| 单片机响应 | `DOCK=OFF,MANUAL` 事件 |
| 副作用 | K2/K3/K4 立即断开; `chargeHostForcedOff=1` |
| 自动判定 | **被暂停** (直到下次 CENTERING 成功) |
| 记录 | `lastRkCmd = LAST_CMD_CHGDIS` |

### 4.7 AT+STOP — 立即停电机

| 触发场景 | 应急停止 |
|---------|----------|
| 单片机响应 | `STOP` 事件 + 蜂鸣器响 200ms |
| 副作用 | X/Y 电机立即停; **不动 K2** |
| 不重置 | `chargeHostManual` / `chargeHostForcedOff` 不变 |

> ⚠️ `AT+STOP` 不动 K2! K2 仍由 CHGENA/CHGDIS/自动判定/保护规则管理.

### 4.8 AT+CONSERVATIVE / AT+RESUME — 保守模式控制

| 命令 | 副作用 |
|------|--------|
| `AT+CONSERVATIVE` | 立即停电机 + 断 K2, 设 `conservative=1`, 发 `CONSERVATIVE,CMD` |
| `AT+RESUME` | 清 `conservative`, 设 `IDLE_CLOSED`, 发 `RESUMED` + `STATE,GOTO=IDLE_CLOSED` |

### 4.9 AT+RKPROT / AT+BMSPROT — 保护策略切换

| 命令 | 默认 | 切换后 |
|------|------|--------|
| `AT+RKPROT=0` | ✓ | P7 链路丢失仅告警, 不脱扣 |
| `AT+RKPROT=1` | | P7 链路丢失进入保守模式 |
| `AT+BMSPROT=0` | ✓ | BMS 离线仅告警 |
| `AT+BMSPROT=1` | | BMS 离线立即脱扣 K2 |

### 4.10 AT+HB — 心跳 (推荐 2Hz)

| 触发场景 | 周期保活, 防止 P7 链路丢失 |
|---------|--------------------------|
| 单片机响应 | (无副作用, 仅刷新 RX 时间戳) |
| 建议频率 | 2Hz (间隔 500ms), 必须 < 10s |
| 自发自收 | `$GARAGE` 推送也算链路存活, 可不发心跳 |

### 4.11 AT+FAULTLOG? — 读故障日志

| 触发场景 | 调试, 历史查询 |
|---------|---------------|
| 单片机响应 | `FAULTLOG,DUMPED` 事件 + 多条 `$FAULT,...` (最多 32 条) |
| 存储 | EEPROM 环形 buffer (每条 7 字节) |
| 编码 | CODE=0x10 (OVER_VOLT), VAL=raw ADC, TS=ms |

### 4.12 AT+FAULTLOG=CLEAR — 清故障日志

| 触发场景 | 调试后清理 |
|---------|-----------|
| 单片机响应 | `FAULTLOG,CLEARED` 事件 |
| 副作用 | EEPROM 写 0xFF × 32 |

### 4.13 AT+SELFTEST — 9 步故障注入

| 触发场景 | 产线下线测试, 现场调试 |
|---------|----------------------|
| 流程 | 9 步, 每步 1.5s, 共 ~13.5s |
| 步骤 | P1..P9 见 §6 保护规则 |
| 响应 | `SELFTEST,BEGIN` + `STEP=P1..P9` + `DONE` |
| 中止 | `AT+SELFTEST=ABORT` → `SELFTEST,ABORT` + 立即清 ALED 状态 |

### 4.14 AT+ALARM=OFF — 立即关蜂鸣器 (新增)

| 触发场景 | 蜂鸣器意外卡响, 应急静音 |
|---------|------------------------|
| 单片机响应 | `$EVT,ALARM,FORCE_OFF` 事件 |
| 副作用 | 清 ALED 状态 + `IO_WriteBit(ALARM, 0)` 强制拉低 |
| 优先级 | 立即生效, 不受 ENABLE_BEEP_* 配置影响 |
| 推荐 | 替代 libapp 的 `AT+ALARM=0` (后者被 ALED_NEG 状态机覆盖) |

### 4.15 AT+OTA / AT+FWU — 固件升级

| 命令 | 用途 |
|------|------|
| `AT+OTA` | 双 Bank IAP 升级 (C bank 已有有效固件时立即跳转) |
| `AT+FWU` | YModem 串口升级 (配合上位机) |

### 4.16 AT+DO / AT+AO — 直写 DO/DAC (危险!)

> ⚠️ **仅限调试**: 这两个命令绕过状态机直接操作 GPIO, 可能破坏机库安全状态!

| 命令 | 副作用 |
|------|--------|
| `AT+DO=<n>,<v>` | 写 DO (n=1..16, v=0/1) — 不影响状态机 |
| `AT+AO1=<v>` | DAC1 输出 (0..4095) |
| `AT+AO2=<v>` | DAC2 输出 (0..4095) |

## 5. P7 链路保护 (Link Watch)

### 5.1 设计理念

**核心机制**: MCU 通过检测 UART1 RX 字节的时间戳判断 RK3568 链路健康度. 任何 UART1 字节 (包括 MCU 自发的 `$GARAGE`) 都会刷新时间戳.

```
                    Host                     MCU
                     │                       │
                     │  AT+HB (2 Hz)         │
                     ├──────────────────────►│  refresh lastRkRxMs
                     │                       │
                     │  $GARAGE (2 Hz)       │
                     │◄──────────────────────┤  self-TX refreshes
                     │                       │  lastRkRxMs too
                     │                       │
              t < 10s│                       │
              (boot  │                       │
              grace) │                       │
                     │                       │
                     │  AT+CHGENA            │
                     ├──────────────────────►│  lastRkCmd=CHGENA
                     │                       │
                     │ ── link lost ──       │
                     │                       │  now - lastRkRxMs > 10s
                     │                       │  rkLinkLostDropEn=0 (default):
                     │  $EVT,RK_LOST,IGNORED │  only log
                     │◄──────────────────────┤  rkLinkLostDropEn=1:
                     │                       │  stop motors + drop K2
                     │                       │
                     │  any byte             │
                     ├──────────────────────►│  link restored
                     │                       │  if conservative:
                     │  $EVT,RK_RESTORED     │  replay lastRkCmd
                     │◄──────────────────────┤
                     │  $EVT,STATE,GOTO=...  │
                     │◄──────────────────────┤
```

### 5.2 三种工作模式

#### 模式 A: 仅监听 (最轻量)

适用: 仅需显示状态, 不发命令的观察节点.

```
RK3568 host 端只读 $GARAGE/$EVT/$FAULT, 不发任何 AT 命令
MCU 自发 $GARAGE 也刷新 lastRkRxMs → 永远不会触发 RK_LOST
```

#### 模式 B: 2Hz 心跳 (推荐, 高可靠)

适用: 命令控制节点 (mission_planner).

```python
import serial, time
ser = serial.Serial('/dev/ttyUSB0', 115200)

while True:
    ser.write(b'AT+HB\r\n')  # 2Hz 心跳
    time.sleep(0.5)
```

#### 模式 C: 命令式 (无心跳)

适用: 一次性调试, 手动命令.

```
> AT+OPEN                    # 命令自身也算心跳
< $EVT,STATE,GOTO=OPENING
... (等 30s) ...
> AT+CLOSE                   # 又一次心跳
< $EVT,STATE,GOTO=CLOSING
```

> ⚠️ **不要**把心跳间隔设到 > 10s, 否则 P7 会误判链路丢失.

## 6. 9 条保护规则 (Protection Rules P1-P9)

> 所有保护规则在 MCU 端独立闭环, **不依赖上位机**. 保护触发时: 立即脱扣 K2 + 停电机 + 蜂鸣器 + `$EVT` 推送 + EEPROM 故障日志.

### 6.1 故障码表

| CODE | 名称 | 中文 | 触发条件 |
|------|------|------|----------|
| `0x01` | `FAULT_EMG_PRESSED` | 急停按下 | DI1=0 |
| `0x02` | `FAULT_EMG_RELEASED` | 急停释放 | 3s lockout |
| `0x10` | `FAULT_OVER_VOLT` | 充电过压 | AI1>3442 (~16.8V) |
| `0x11` | `FAULT_OVER_CURR` | 充电过流 | AI2>3686 (~18mA) ≥1s |
| `0x12` | `FAULT_FULL_NO_DROP` | 满电未降流 | V≥3378 (~16.5V) AND I≥3276 (~16mA) ≥5s |
| `0x20` | `FAULT_COVER_TIMEOUT` | 舱盖超时 | X 电机 >30s 未到限位 |
| `0x21` | `FAULT_CENTER_TIMEOUT` | 归中超时 | Y 归中 >15s 未到中心 |
| `0x22` | `FAULT_RELEASE_TIMEOUT` | 释放超时 | Y 释放 >15s 未到外侧 |
| `0x23` | `FAULT_LIMIT_INVALID` | 限位异常 | 状态 vs DI 不一致 |
| `0x30` | `FAULT_RK_LOST` | RK 链路丢失 | 10s UART 静默 |
| `0x40` | `FAULT_POWER_UNDER` | 电源欠压 | (预留) |

### 6.2 P1: 电机超时 (Motor Timeout)

**目的**: 防止机械卡死时电机持续运行烧毁.

| 子规则 | 阈值 | 动作 |
|--------|------|------|
| 舱盖开/关 | 30s 未到限位 | 停电机 + FAULT_COVER_TIMEOUT + safe-fail |
| 归中 | 15s 未到中心 | 停电机 + FAULT_CENTER_TIMEOUT + 回 IDLE_OPEN |
| 释放 | 15s 未到外侧 | 停电机 + DropCharge + FAULT_RELEASE_TIMEOUT + 回 IDLE_OPEN |

### 6.3 P2: 限位不一致 (Limit Switch Inconsistency)

**目的**: 检测限位开关与状态机是否矛盾 (如开盖时却检测到关限位).

| 状态 | 不一致条件 | 动作 |
|------|------------|------|
| OPENING | `coverAtClosed==1` | 停电机 + FAULT_LIMIT_INVALID + 回 IDLE_OPEN |
| CLOSING | `coverAtOpen==1` | 停电机 + FAULT_LIMIT_INVALID + 回 IDLE_CLOSED |
| CENTERING | 任一板在外侧 | 停电机 + FAULT_LIMIT_INVALID |
| RELEASING | 任一板在中心 | 停电机 + FAULT_LIMIT_INVALID |

### 6.4 P3: 急停-运行不一致 (E-stop / Run-time Conflict)

**目的**: 检测 EMG 已按下但电机仍在运行.

**条件**: `emgLocked=1 AND motor running`

**动作**: 立即停电机 + DropCharge + EMG_OUT_RELAY (K1) ON + FAULT_EMG_PRESSED.

> 注: 当前实现是 EMG 触发时立即 `EMG_LockNow()` 停电机, P3 与 P9 行为相同.

### 6.5 P4: 充电过压 (Over Voltage)

**阈值**: AI1 raw > 3442 (~16.8V)

**动作**: DropCharge + FAULT_OVER_VOLT + `$EVT,OVER_VOLT,V=high`.

> ⚠️ **手动不能关闭**: 即使 `AT+CHGENA`,P4 仍会立即脱扣.

### 6.6 P5: 充电过流 (Over Current)

**阈值**: AI2 raw > 3686 (~18mA) 持续 ≥ 1s (有 1s 保持时间, 避免噪声误触发).

**动作**: DropCharge + FAULT_OVER_CURR + `$EVT,OVER_CURR,I=high`.

### 6.7 P6: 满电未降流 (Full Voltage No Drop)

**目的**: 检测充电器硬件故障 — 满电后电流应降为 0, 若仍保持高位则异常.

**阈值**: V≥3378 (~16.5V) AND I≥3276 (~16mA) 持续 ≥ 5s.

**动作**: DropCharge + FAULT_FULL_NO_DROP + `$EVT,FULL_VOLT_NO_DROP,V,I`.

### 6.8 P7: RK3568 链路丢失 (Link Lost)

**阈值**: 10s 无任何 UART1 字节.

**默认策略** (`rkLinkLostDropEn=0`): **仅告警** — `$EVT,RK_LOST,IGNORED`.

**严格策略** (`AT+RKPROT=1`): 进入保守模式 — 停电机 + DropCharge + `$EVT,CONSERVATIVE,RK_LOST`.

**恢复**: 任意 UART 字节 → `$EVT,RK_RESTORED` + 重放 `lastRkCmd`.

### 6.9 P8: 电源欠压 (Power Undervoltage)

**状态**: 预留, 当前未实现.

### 6.10 P9: 急停按下 (E-stop Pressed)

**条件**: DI1 = 0 (急停按下).

**动作**: 立即 `EMG_LockNow()` — 停电机 + DropCharge + EMG_OUT_RELAY (K1) ON + FAULT_EMG_PRESSED + `$EVT,EMG=PRESSED`.

**释放**: DI1=1 持续 3s (`EMG_LOCKOUT_HOLD_MS`) → 解锁 + `$EVT,EMG=RELEASED` + EMG_OUT_RELAY OFF.

### 6.11 与手动命令的关系

| 场景 | 手动命令效果 | 保护规则效果 |
|------|--------------|--------------|
| `AT+CHGENA` 后发生过压 | 已接通 K2 | **P4 立即脱扣**, 手动无效 |
| `AT+CHGENA` 后急停 | 已接通 K2 | **P9 立即脱扣**, 手动无效 |
| `AT+CHGENA` 后电机超时 | 已接通 K2 | **P1 立即停电机**, K2 状态不变 (保护规则不脱扣 K2, 除非另有触因) |
| `AT+STOP` 后发生过压 | K2 仍 ON | **P4 立即脱扣**, K2 OFF |

### 8.1 /hangar/state — GarageState.msg

**字段填充规则**:

## 11. 集成示例 (Integration Examples)

### 11.1 Python: 命令下发节点

```python
import rclpy
from rclpy.node import Node
from hangar_msgs.msg import GarageCommand, GarageState

class HangarClient(Node):
    def __init__(self):
        super().__init__('hangar_client')
        self.pub = self.create_publisher(GarageCommand, '/hangar/cmd', 10)
        self.sub = self.create_subscription(
            GarageState, '/hangar/state', self._on_state, 10)

    def _on_state(self, msg: GarageState):
        if msg.state == GarageState.STATE_PLATES_CENTER and not msg.conservative:
            self.get_logger().info('hangar ready, drone can dock')
        if msg.conservative:
            self.get_logger().warn('hangar in conservative mode, abort mission')

    def open_cover(self):
        msg = GarageCommand()
        msg.cmd = GarageCommand.CMD_OPEN
        msg.detail = 'mission start'
        self.pub.publish(msg)


def main(args=None):
    rclpy.init(args=args)
    node = HangarClient()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 11.2 Python: 命令语义建议

| 场景 | 步骤 |
|------|------|
| **起飞前** | 1. `CMD_CHG_DISABLE` (确保 K2 OFF)<br>2. 等待 `/hangar/fault` 中 `DOCK=OFF`<br>3. `CMD_RELEASE`<br>4. 等待 `/hangar/state.state == STATE_IDLE_OPEN`<br>5. `CMD_CLOSE`<br>6. 等待 `STATE_IDLE_CLOSED` |
| **充电流程** | 1. `CMD_OPEN` (开舱)<br>2. 等待 `STATE_IDLE_OPEN`<br>3. `CMD_CENTER` (归中)<br>4. 等待 `STATE_PLATES_CENTER`<br>5. **不**发 `CMD_CHG_ENABLE` — 让 MCU 自动判定<br>6. 监控 `/hangar/state.dock_on` 切换 |
| **应急停止** | `CMD_STOP` — 仅停电机, 不动 K2 |
| **退出保守模式** | `CMD_RESUME` — 退出 P7 + 回 IDLE_CLOSED |
| **查故障** | `CMD_FAULTLOG_QUERY` — 等 1s, 读 `/hangar/fault` 多帧 |

### 11.3 C++ ROS2 节点 (Publisher)

```cpp
#include "rclcpp/rclcpp.hpp"
#include "hangar_msgs/msg/garage_command.hpp"

class HangarPub : public rclcpp::Node {
public:
    HangarPub() : Node("hangar_pub") {
        pub_ = this->create_publisher<hangar_msgs::msg::GarageCommand>(
            "/hangar/cmd", 10);
    }

    void open_cover() {
        auto msg = hangar_msgs::msg::GarageCommand();
        msg.cmd = hangar_msgs::msg::GarageCommand::CMD_OPEN;
        msg.detail = "cpp open";
        pub_->publish(msg);
    }

private:
    rclcpp::Publisher<hangar_msgs::msg::GarageCommand>::SharedPtr pub_;
};
```

### 11.4 与 MAVROS 配合 (参考)

> 本协议**不**直接耦合 MAVROS. MAVROS 与机库通过 ROS2 topic 桥接:

```
                 MAVROS                          hangar_bridge
                    │                                  │
   mavros/state ────┤                                  │
   mavros/battery ──┼──► [mission_planner] ──► /hangar/cmd
   mavros/...    ───┤              │                    │
                    │              └─── /hangar/state ───┤
                    │                  /hangar/fault ────┤
```

一个示例的 mission_planner 节点:
- 监听 `mavros/state` (FC mode)
- 收到 `MODE_AUTO_MISSION` + `BATTERY=15%` → `CMD_OPEN` → `CMD_CENTER`
- 监听 `/hangar/state.state == STATE_PLATES_CENTER` → 通过 MAVROS 触发 LAND 模式
- 监听 `mavros/battery` 充满 (`BATTERY=100%`) → 准备 `CMD_RELEASE` + `CMD_CLOSE`

## 12. 典型场景 (Typical Scenarios)

### 12.1 场景 A: 单 RK3568 本地调试 (基础)

**硬件**: 1 台 RK3568, USB 接 STM32.

**步骤**:
1. RK3568 启动 Ubuntu + ROS2 + hangar_bridge
2. 终端 1: `ros2 topic echo /hangar/state` (监控状态)
3. 终端 2: `ros2 topic pub --once /hangar/cmd hangar_msgs/GarageCommand "{cmd: 1}"` (开舱)
4. 观察 `$GARAGE,State` 从 IDLE_CLOSED → OPENING → IDLE_OPEN

### 12.2 场景 B: 多 ROS2 节点 (推荐生产配置)

**硬件**:
- RK3568 (192.168.2.44): 物理接 STM32, 跑 hangar_bridge
- PC-甲 (任意 IP): ROS2 节点, 仅订阅状态
- PC-乙 (任意 IP): ROS2 节点, 仅发命令

**要求**: 所有 PC 在同一 ROS2 domain (`ROS_DOMAIN_ID` 相同).

**部署**:
1. RK3568 启动 hangar_bridge (默认 `ROS_DOMAIN_ID=0`)
2. PC-甲 `export ROS_DOMAIN_ID=0; ros2 topic echo /hangar/state` → 应能收到
3. PC-乙 `export ROS_DOMAIN_ID=0; ros2 topic pub --once /hangar/cmd ...` → 应能发命令

### 12.3 场景 C: 起飞前完整流程

```
1. 无人机着陆, BMS SOC=10%
2. ROS2 mission_planner 监听 /hangar/state
3. mission_planner 发 CMD_OPEN
4. 等待 STATE=IDLE_OPEN
5. mission_planner 发 CMD_CENTER
6. 等待 STATE=PLATES_CENTER, dock_on=1 (自动判定)
7. 充电中: SOC 监控 + 充电过流保护
8. SOC=100% → 自动 DOCK=OFF (AUTO_FULL)
9. 无人机准备起飞 → mission_planner 发 CMD_RELEASE
10. 等待 STATE=IDLE_OPEN
11. mission_planner 发 CMD_CLOSE
12. 等待 STATE=IDLE_CLOSED
```

### 12.4 场景 D: 紧急急停

```
1. 现场人员按下急停 (DI1=0)
2. MCU 立即 EMG_LockNow()
3. MCU 推 $EVT,EMG=PRESSED → bridge 转 /hangar/fault.code=FAULT_EMG_PRESSED
4. MCU 停电机 + DropCharge + EMG_OUT_RELAY ON
5. ROS2 mission_planner 收到 fault → 立即取消后续动作
6. 现场人员释放急停 (DI1=1)
7. MCU 等 3s lockout → 推 $EVT,EMG=RELEASED
8. MCU 解锁 + EMG_OUT_RELAY OFF
9. ROS2 mission_planner 发 CMD_RESUME + CMD_OPEN (重启流程)
```

### 12.5 场景 E: RK3568 链路丢失与恢复

```
1. RK3568 ROS2 节点死机 / 网线断开
2. MCU 10s 内无 UART 字节 → 触发 P7
3. 默认策略 rkLinkLostDropEn=0: 仅 $EVT,RK_LOST,IGNORED
4. 严格策略 rkLinkLostDropEn=1: 停电机 + DropCharge + $EVT,CONSERVATIVE,RK_LOST
5. RK3568 恢复 → 任意 UART 字节
6. MCU 推 $EVT,RK_RESTORED
7. 若处于 conservative: MCU 重放 lastRkCmd → 推 $EVT,STATE,GOTO=...
8. 若仅 IGNORED: RK3568 须主动重发 OPEN/CLOSE 等命令
```

## 13. 故障排查 (Troubleshooting)

### 13.1 /hangar/state 完全没消息

| 排查项 | 方法 |
|--------|------|
| 串口路径对吗? | `ls /dev/ttyUSB*` (Linux) / 设备管理器 (Windows) |
| CH340 驱动加载? | `dmesg \| grep ch340` (Linux) |
| 波特率对吗? | 确认 115200 |
| 串口被独占? | `lsof /dev/ttyUSB0` (Linux), 关闭其他串口程序 |
| 板子有电? | 测 5V/12V 输入 |
| bridge 启动日志? | `ros2 node info /hangar_bridge` |

### 13.2 /hangar/state 有消息但 state 一直不变

| 排查项 | 方法 |
|--------|------|
| 命令发了没? | `ros2 topic echo /hangar/cmd` 看下游是否收到 |
| MCU 是否收到? | 直接串口发 `AT+OPEN` 看是否回 `$EVT,STATE,GOTO=OPENING` |
| 状态机卡在哪? | 看 `$GARAGE,State` 和 `$GARAGE,Cover` 字段 |
| 急停按下了? | 看 `EMG=PRESSED` 事件 |
| 保守模式? | 看 `Conservative=YES` 字段 |

### 13.3 /hangar/fault 频繁 RK_LOST

| 排查项 | 方法 |
|--------|------|
| 心跳太慢? | bridge 默认 2Hz, 检查 `_on_heartbeat()` |
| 自发自收停? | `$GARAGE` 推送也算链路, 不发心跳也行 |
| 串口物理断? | 测 UART1 TX/RX 波形 |
| RK3568 ROS2 节点死? | `ros2 node list` |

### 13.4 volt_raw/curr_raw=0 但 dock_on=1

**原因**: `state` 不在 `PLATES_CENTER`. ADC 采样仅在该状态下刷新.

**解决**: 等状态机迁移到 `PLATES_CENTER`.

### 13.5 $EVT,OVER_VOLT 一直出

**原因**: 电压阈值 `V_OVER_RAW=3442 (~16.8V)` 与实际充电器输出不匹配.

**解决**: 用万用表实测充电电压, 调整 `V_OVER_RAW` 在 `source/user_app.c:150`.

### 13.6 AT+CHGENA 没生效

**可能原因**:
- `state=PLATES_CENTER && bms_soc>=100%` → 自动判定已脱扣
- 处于 P4/P5/P6 保护中 (K2 已被强制脱扣)
- 处于 P9 急停 (emgLocked=1)
- 处于 P7 保守模式 (conservative=1)

**解决**: 检查 `/hangar/state.state`、`conservative` 字段, 以及 `/hangar/fault` 历史.

### 13.7 上电后 10s 内 RK_LOST 不出现

**这是设计**: boot grace 期内静默 (`Protection_RkLink` line 758 early-return).

**解决**: 等 10s 之后.

### 13.8 蜂鸣器一直响 (老固件 V1.30 已知 bug)

**原因**: ALED_NEG 状态机 IDLE 不写低电平, 引脚卡 HIGH.

**解决**: 升级到 V1.31.02, 或手动发 `AT+ALARM=OFF`.

### 13.9 SELFTEST=ABORT 后蜂鸣器响

**原因 (V1.30 老固件)**: ABORT 路径没清 ALED 状态.

**解决 (V1.31.02 已修复)**: 升级后 ABORT 自动清 ALED; 否则手动 `AT+ALARM=OFF`.

### 13.10 端口冲突

```bash
# Linux: 找占用 ttyUSB 的进程
sudo lsof /dev/ttyUSB0
# 杀掉: sudo kill -9 <pid>
```

### 13.11 ROS2 discovery 失败 (多 PC)

| 排查项 | 方法 |
|--------|------|
| `ROS_DOMAIN_ID` 一致? | 所有 PC 必须相同 |
| 防火墙挡了 UDP? | ROS2 用 UDP multicast, 需开放 7400-7500 端口 |
| DDS 实现不同? | 默认 Fast DDS, 跨网段可能需调 |

## 附录 C: 故障处理矩阵

| 故障场景 | UART 协议层响应 | ROS2 topic 响应 |
|---------|----------------|-----------------|
| 上位机发 `AT+CHGENA` 后断开 | 10s 后 `$EVT,RK_LOST,IGNORED` | `/hangar/fault.code=FAULT_RK_LOST` |
| 上位机发 `AT+CHGENA` 后 BMS 满电 | 自动 `$EVT,DOCK=OFF,AUTO_FULL` | `/hangar/state.dock_on=0` |
| 上位机发 `AT+CHGENA` 后过压 | 立即 `$EVT,OVER_VOLT,V=high` + DropCharge | `/hangar/fault.code=FAULT_OVER_VOLT` |
| 急停按下 | `$EVT,EMG=PRESSED` + 停电机 | `/hangar/fault.code=FAULT_EMG_PRESSED` |
| 急停松开 | `$EVT,EMG=RELEASED` (3s 后) | `/hangar/fault.code=FAULT_EMG_RELEASED` |
| 舱盖开超时 | `$EVT,COVER_OPEN_TIMEOUT` | `/hangar/fault.code=FAULT_COVER_TIMEOUT` |
| 归中超时 | `$EVT,CENTER_TIMEOUT` | `/hangar/fault.code=FAULT_CENTER_TIMEOUT` |
| 限位异常 | `$EVT,LIMIT_INVALID,<DETAIL>` | `/hangar/fault.code=FAULT_LIMIT_INVALID` |
| **bridge 死机** | (无影响, MCU 仍推 $GARAGE) | `/hangar/state` 停止 |
| **RK3568 整体死机** | MCU 自发 $GARAGE 仍刷新 link | host 重启后自动恢复 |
| **蜂鸣器卡响 (老固件)** | `$EVT` 仍正常 | (无 topic 影响) |
| **蜂鸣器卡响 (V1.31.02)** | 已修复 | (无此问题) |
